JDG translator notes

noted issues:

- boolean attributes: JDG will throw an exception if no value is specified on the insert or

no default value is defined in the protobuf definition file.

- char data types: Protobuf data types do not support char type's (see: for list)

- will not push down NULL (Is Null/Is Not Null) criteria, or <> / Not criteria, this is because

JDG considers NULL <> 1 = true (and SQL does not)

1. (not suppose to be in DV 6.2) Operator <> incorrectly handled

- teiid-3573

- https://bugzilla.redhat.com/show_bug.cgi?id=1242999



2. (not in DV 6.2) incorrect null ordering

- teiid-3570

https://bugzilla.redhat.com/show_bug.cgi?id=1242938



3. cannot delete/update rows inserted from other hotrod client

- TEIID-3536

https://bugzilla.redhat.com/show_bug.cgi?id=1232282



To resolve this, the CacheTypeMap property will be enhanced. The current format is:

cacheName:className[;pkFieldName]

Changing it so that the object type of the "pkFieldName" can be specified. So changing the format to:

cacheName:className[;pkFieldName[:cacheKeyJavaType]]

And only when specified, the key value will be transformed to this type before doing a lookup by key (i.e, for updates and deletes) or insert.





4. Infinispan-dsl-cache translator: can't insert value into BYTE and BIGINTEGER columns

- TEIID-3533

https://bugzilla.redhat.com/show_bug.cgi?id=1232182



- ideally, either use the Teiid connection importer so that the data types are set accordingly, or use

a dynamic vdb that only maps the translator type (infinispan-cache-dsl) and jndi name to the data source/resource adapter
